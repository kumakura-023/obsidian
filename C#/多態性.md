```C#
class A {
    public virtual void Speak() => Console.WriteLine("I am A");
}

class B : A {
    public override void Speak() => Console.WriteLine("I am B");
}
```

```C#
A classA = new B();
classA.Speak(); // => "I am B"
```
見た目は「A型の変数」だけど、中身は `B`。  
Aが持つメソッドを呼んだとき、中身（B）の実装が動く。  
これが「**多態性 (polymorphism)**」の正体。

### 🔹なぜこんなことをするのか？

アップキャストを使うと、「**共通の型でまとめて扱える**」という利点が出る。
```C#
`List<A> list = new List<A> {     
	new A(),
	new B(),
	new B() 
	};  
	foreach (var item in list)
		item.Speak(); // それぞれ A/B に応じた挙動をする！`
```


つまり、Aという共通インターフェイスで扱いつつ、  
実際の動作は“中身の型に応じて変わる”。  
これがポリモーフィズムの力。
